# Export review reference

## File roles (`tests/export/`)

| File | Role |
|------|------|
| `*.org` | Hand-written fixtures (checked in) |
| `*.json` | Snapshot tests via `tests/test-export.el`; `make update-exports` |
| `*.md` | Agent/human review view from JSON; gitignored; `make update-exports-pretty` |
| `*.html` | Optional HTML view; same regeneration as `.md` |

Coverage checklist: `planning/elements.md`.

Deterministic export options: `tests/ox-json-test-helpers.el` (`:author`, `:date`, `:email`, `:json-strict t`, `:json-deterministic-refs t`).

## `.md` format

Not a second export format. Generated by `scripts/json-to-markdown.py` from `.json`.

Structure mirrors Org headline hierarchy:

1. **Document** — `# Title`, then “File properties:” + one JSON block (`org-document`, no `contents`).
2. **Each headline** — `##` / `###` / … (markdown level = Org level + 1). Title from `raw-value` or parsed `title`.
3. **Under each headline**, in order:
   - **Headline properties:** — full `headline` node JSON (no inlined `contents`).
   - **Section properties:** — first child `section` node if present.
   - **Section contents:** — one JSON block per top-level body element (paragraph, plain-list, table, …), each with its own `contents` tree.

Nested headlines repeat the same pattern under deeper markdown headings.

### Limitations

- One JSON object per “Section contents” block; deep nesting lives in that object's `contents`.
- Document/headline blocks are shallow; section body is split into separate blocks.
- Plain string fragments at section level appear as JSON strings; inline text stays inside paragraph nodes.

Companion: `scripts/json-to-html.py` renders the same JSON as nested HTML cards.

## Common failure patterns

| Symptom | Likely cause |
|---------|----------------|
| Element missing entirely | Fixture syntax or exporter skip |
| Planning on section body, not headline props | Org limitation for combined DEADLINE/SCHEDULED/CLOSED |
| `#+description:` empty in doc props | Exporter / keyword handling |
| Comment block, CLOCK, LOGBOOK wrong | Often exporter gaps |
| Babel `:exports results` shows result node | Expected Org export behavior |
| `#+TBLFM:` not on table node | Org attaches formula separately |
| Priority `[#10]` missing | Org/config, not fixture typo |

When fixing fixtures, re-run `make update-exports` and `make test` before updating `elements.md`.
